# 📦 Instalação e Uso - IGRP CLI Next

Este guia mostra como instalar e usar o **IGRP CLI Next** - um detector avançado de controle de acesso para projetos Next.js.

## 🚀 Instalação

### 1. Instalação Global (Recomendado)

```bash
# Usando npm
npm install -g igrp-cli-next

# Usando yarn
yarn global add igrp-cli-next

# Usando pnpm
pnpm add -g igrp-cli-next

# Ou usando o script de instalação
chmod +x install-global.sh && ./install-global.sh
```

### 2. Instalação Local

```bash
# Instalar como dependência de desenvolvimento
npm install --save-dev igrp-cli-next

# Ou como dependência regular
npm install igrp-cli-next
```

### 3. Verificar Instalação

```bash
# Verificar se foi instalado corretamente
npx igrp-cli-next --help

# Ou se instalado globalmente
igrp-cli-next --help
```

## 🔧 Configuração Inicial

### 1. Configurar Scripts no package.json

Adicione estes scripts ao seu `package.json`:

```json
{
  "scripts": {
    "detect": "igrp-cli-next",
    "detect:verbose": "igrp-cli-next --verbose",
    "detect:save": "igrp-cli-next --output access-config.json",
    "detect:menu": "igrp-cli-next --generate-menu",
    "detect:full": "igrp-cli-next --generate-menu --output access.json --verbose"
  }
}
```

### 2. Configurar Variáveis de Ambiente (Opcional)

Crie um arquivo `.env.local`:

```bash
# API de gerenciamento de acesso (opcional)
ACCESS_MANAGEMENT_API_URL=https://your-api.com/access-management
ACCESS_MANAGEMENT_API_KEY=your-api-key-here
```

## 📝 Configurar Controle de Acesso nas Páginas

### 1. Página Básica (Pública)

```typescript
// app/page.tsx ou pages/index.tsx
export const accessConfig = {
  protected: false,
  title: "Página Inicial",
  description: "Página inicial pública do site",
  permissions: ["read"]
};

export default function HomePage() {
  return <div>Página inicial</div>;
}
```

### 2. Página Protegida

```typescript
// app/dashboard/page.tsx
export const accessConfig = {
  protected: true,
  title: "Dashboard",
  description: "Dashboard principal do usuário",
  permissions: ["dashboard:read", "dashboard:write"],
  menu: true, // Incluir no menu
  version: "1.0.0",
  metadata: {
    requiresAuth: true,
    cacheTimeout: "5m"
  }
};

export default function DashboardPage() {
  return <div>Dashboard</div>;
}
```

### 3. Rota Dinâmica

```typescript
// app/users/[id]/page.tsx
export const accessConfig = {
  protected: true,
  title: "Perfil do Usuário",
  description: "Página de perfil do usuário",
  permissions: ["user:read", "user:update"],
  menu: true
};

export default function UserProfilePage({ params }: { params: { id: string } }) {
  return <div>Perfil do usuário {params.id}</div>;
}
```

### 4. Rota Catch-all

```typescript
// app/blog/[...slug]/page.tsx
export const accessConfig = {
  protected: false,
  title: "Blog",
  description: "Páginas do blog",
  permissions: ["blog:read"],
  menu: true
};

export default function BlogPage({ params }: { params: { slug: string[] } }) {
  return <div>Blog: {params.slug.join('/')}</div>;
}
```

### 5. Padrões de Permissão Suportados

O plugin suporta múltiplos padrões de permissão:

```typescript
// Padrão Zanzibar (recomendado)
permissions: ["user:create", "post:read", "comment:update"]

// Padrão CRUD
permissions: ["create_user", "read_post", "update_comment"]

// Padrão Resource
permissions: ["user.create", "post.read", "comment.update"]

// Padrão Custom
permissions: ["admin_access", "moderator", "viewer"]
```

## 🎯 Executar o Detector

### 1. Execução Básica

```bash
# Execução simples
npx igrp-cli-next

# Com saída detalhada
npx igrp-cli-next --verbose

# Salvar em arquivo
npx igrp-cli-next --output access-config.json

# Gerar menu
npx igrp-cli-next --generate-menu
```

### 2. Execução Avançada

```bash
# Detecção completa com menu e API
npx igrp-cli-next --generate-menu --send-api --verbose

# Sem validação (mais rápido)
npx igrp-cli-next --no-validation --no-tree

# Projeto específico
npx igrp-cli-next --project-root ./packages/my-app

# Enviar para API customizada
npx igrp-cli-next --send-api --api-endpoint https://api.example.com
```

### 3. Usando Scripts do package.json

```bash
# Executar detector básico
npm run detect

# Executar com saída detalhada
npm run detect:verbose

# Salvar configuração
npm run detect:save

# Gerar menu
npm run detect:menu

# Execução completa
npm run detect:full
```

## 🔧 Configuração Avançada

### 1. Suporte a Monorepos

O plugin funciona automaticamente em monorepos:

```bash
# Em um monorepo
cd packages/my-app
npx igrp-cli-next

# Ou especificar o projeto
npx igrp-cli-next --project-root packages/my-app
```

### 2. Integração com API

```bash
# Enviar dados para API
npx igrp-cli-next --send-api

# API customizada
npx igrp-cli-next --send-api --api-endpoint https://api.example.com --api-key your-key
```

### 3. Validação de Permissões

```bash
# Validar apenas permissões
npx igrp-cli-next --validate-permissions --verbose

# Desabilitar validação
npx igrp-cli-next --no-validation
```

## 🏗️ Arquitetura

O plugin foi construído com uma arquitetura modular:

- **Core Detector**: Lógica principal de escaneamento e processamento
- **Utility Classes**: Componentes especializados para tarefas específicas
  - `ProjectUtils`: Detecção de projeto e informações do package
  - `ConfigParser`: Parsing e validação de configurações de acesso
  - `RouteConverter`: Conversão de rotas e extração de parâmetros
  - `PermissionValidator`: Validação de padrões de permissão
  - `VersionUtils`: Verificação e comparação de versões
  - `SummaryGenerator`: Geração de relatórios e estatísticas
  - `APIClient`: Integração com APIs
  - `MenuGenerator`: Geração de configurações de menu
