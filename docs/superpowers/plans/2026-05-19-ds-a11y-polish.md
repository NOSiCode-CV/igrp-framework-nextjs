# Design System A11y & Polish Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix accessibility violations, `size-*` rule violations, bundle bloat, stale props, and typography polish uncovered in a frontend-design / shadcn / web-interface-guidelines review of the promoted Horizon components.

**Architecture:** Pure fix pass — no new components, no new props except `onError` on `IGRPImageCropper`. Every change is a targeted correction to an existing file; no refactors beyond what is named.

**Tech Stack:** React 19, TypeScript, Tailwind v4, CVA, Radix UI, pnpm monorepo (`packages/design-system`)

- [ ] **Step 3: Fix `role="link"` on `IGRPBreadcrumbPage`**

`role="link" aria-disabled="true"` on a `<span>` is semantically wrong — a link that is permanently disabled is just not a link. The current-page breadcrumb item should be a plain span with only `aria-current="page"`.

Find `IGRPBreadcrumbPage` (~line 348). Change:
```tsx
// Before
<span
  ref={ref}
  role="link"
  aria-disabled="true"
  aria-current="page"
  className={cn('font-semibold flex items-center', colorClass, className)}
  {...props}
>

// After
<span
  ref={ref}
  aria-current="page"
  className={cn('font-semibold flex items-center', colorClass, className)}
  {...props}
>
```

Remove both `role="link"` and `aria-disabled="true"`.

- [ ] **Step 1: Add entry animation to both variants**

Tailwind v4 supports `animate-in`, `slide-in-from-bottom-*`, and `fade-in` via `tailwindcss-animate` (already used by Radix primitives in the DS). Add `motion-safe:` guards so reduced-motion users see instant transitions.

  "fixed bottom-0 left-0 right-0 z-50 flex items-center justify-between gap-4 border-t bg-background px-6 py-4 shadow-lg",
