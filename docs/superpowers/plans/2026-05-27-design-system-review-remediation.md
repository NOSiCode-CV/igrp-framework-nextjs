# Design-System Review Remediation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the accessibility, forms, design-token, and reduced-motion findings from the web-interface-guidelines + shadcn review of `@igrp/igrp-framework-react-design-system`.

**Architecture:** All edits live in one package (`packages/design-system`). A canonical shadcn `Spinner` primitive is added and `IGRPLoadingSpinner` is recomposed on top of it. Raw Tailwind colors are replaced with the semantic tokens already defined in `tokens.css`. `prefers-reduced-motion` is honored via additive `motion-reduce:` utilities (which appear in `dist/` and are generated by the consuming app's Tailwind scan — a global CSS reset cannot reach token-only consumers). `transition: all` is narrowed to explicit property lists **only in the horizon/custom layers**; shadcn-mirrored primitives keep `transition-all` to preserve `shadcn add --diff` upstream parity.

**Tech Stack:** React 19, Tailwind v4 (CSS-first, OKLCH tokens), CVA, Radix, lucide-react, SWC+Babel (React Compiler) build, ESLint+Prettier, changesets (pre-release `beta`, always `patch`).

## Testing reality (read before starting)

This package's `vitest.config.ts` uses `environment: "node"` and includes only `src/**/*.test.ts` (not `.tsx`); the sole test is `src/lib/color-utils.test.ts` (pure logic). There is **no jsdom / React Testing Library harness**, and these changes are JSX/className/attribute edits with no extractable pure logic. Per the repo's own convention (`CLAUDE.md`), visual + a11y testing is routed to `packages/design-system-storybook` (Playwright snapshots).

Therefore this plan does **not** use per-component TDD. Each task is verified by: exact `grep` assertions on the edited source, a successful `pnpm build:ds`, and ESLint/Prettier. Storybook snapshot review is a single final task handed off for human eyes. Do **not** add a test harness — it is out of scope and unauthorized.

## Task 2: Recompose `IGRPLoadingSpinner` on `Spinner` with a11y + tokens

**Files:**
- Modify: `packages/design-system/src/components/horizon/loading-spinner.tsx` (full rewrite)

Current file hand-rolls an `animate-spin` div with raw `border-gray-*` colors, manual `dark:`, and `h-12 w-12`, and announces nothing to assistive tech.

- [ ] **Step 1: Replace the component body**

Full new file content:

```tsx
"use client"

import { useId } from "react"

import { cn } from "../../lib/utils"
import { Spinner } from "../primitives/spinner"

/**
 * Props for the IGRPLoadingSpinner component.
 * @see IGRPLoadingSpinner
 */
interface IGRPLoadingSpinnerProps {
  /** CSS classes for the outer container (centers the spinner). */
  parentClassName?: string
  /** CSS classes for the spinner element. */
  className?: string
  /** HTML id attribute. */
  id?: string
  /** Accessible loading label announced to assistive tech. */
  label?: string
}

/**
 * Centered loading spinner for async states.
 * Announces its busy state via role="status" + aria-live.
 */
function IGRPLoadingSpinner({ parentClassName, className, id, label = "Loading…" }: IGRPLoadingSpinnerProps) {
  const _id = useId()
  const ref = id ?? _id

  return (
    <div
      className={cn("flex justify-center items-center h-64", parentClassName)}
      id={ref}
      role="status"
      aria-live="polite"
    >
      <Spinner className={cn("size-12 text-primary", className)} aria-label={undefined} />
      <span className={cn("sr-only")}>{label}</span>
    </div>
  )
}

export { IGRPLoadingSpinner, type IGRPLoadingSpinnerProps }
```

Notes: `role="status"` + the `sr-only` text live on the container so the label is announced once; `aria-label={undefined}` on the inner `Spinner` avoids a duplicate announcement. `size-12` replaces `h-12 w-12`; `text-primary` replaces `border-gray-900 dark:border-gray-100`. `motion-reduce:animate-none` is inherited from the `Spinner` primitive.

## Task 8: Honor `prefers-reduced-motion` on `animate-*` usages (all layers)

Scope: **all layers including primitives** — this is additive (appending one utility), accessibility-critical, and shows as a trivial, conflict-free hunk in `shadcn add --diff`. Target the high-impact motion: every `animate-*` class (spin, `animate-in`/`animate-out` overlay fades/zooms/slides, pulse, bounce, accordion height). `Spinner` (Task 1) and `IGRPLoadingSpinner` (Task 2) already have it.

**Codemod rule:** for every element whose `className` contains an `animate-*` utility (excluding the already-done spinner files), append `motion-reduce:animate-none` to the same class string. Apply by hand per file — do not write a script.

- [ ] **Step 1: Enumerate the targets**

Run (from repo root): grep for `animate-` across `packages/design-system/src/components` (both primitives and horizon/custom). This yields the working list — primarily the overlay primitives (`alert-dialog.tsx`, `dialog.tsx`, `drawer.tsx`, `sheet.tsx`, `popover.tsx`, `dropdown-menu.tsx`, `context-menu.tsx`, `menubar.tsx`, `hover-card.tsx`, `tooltip.tsx`, `select.tsx`, `navigation-menu.tsx`, `command.tsx`, `accordion.tsx`, `collapsible.tsx`), `skeleton.tsx` (`animate-pulse`), `progress.tsx`, `sonner.tsx`, `stepper.tsx`, and any horizon component using `animate-*`.

- [ ] **Step 2: Apply the appended utility per file**

Worked example — `skeleton.tsx`:

```tsx
// before
className={cn("bg-accent animate-pulse rounded-md", className)}
// after

```

Worked example — `dialog.tsx` content (`data-[state=open]:animate-in data-[state=closed]:animate-out ...`): append `motion-reduce:animate-none` once to that class string. For `data-[state=*]:animate-in`/`animate-out` pairs, a single `motion-reduce:animate-none` on the element disables both. Repeat the same edit on each enumerated element. Do **not** alter any non-`animate-*` classes.
