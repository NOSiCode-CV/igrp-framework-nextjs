# Design System Close-Out Bundle A Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close out items 1, 2, 3, and 6 from the design-system close-out spec — finish the `/styles` removal, catalogue the four remaining IGRP-custom primitive deltas, establish a shadcn-drift baseline, and add the Horizon-layer `IGRPMenubar` wrapper.

**Spec:** [docs/superpowers/specs/2026-05-28-design-system-closeout-spec.md](docs/superpowers/specs/2026-05-28-design-system-closeout-spec.md). Items 4 (docs site) and 5 (component tests) are intentionally out of scope — they belong in separate plans.

## Task C: Establish the shadcn-drift baseline

**Spec reference:** Item 3. **Depends on Task B** — run only after the deltas catalogue is complete.

**Files:**
- Create: `packages/design-system/scripts/drift-baseline-2026-05.md` (the snapshot report).
- Possibly modify: any primitive where (a) upstream drift is to be absorbed, or (b) the `// shadcn: YYYY-MM-DD` stamp needs refreshing because the divergence is now documented.
- Possibly modify: `packages/design-system/scripts/check-shadcn-drift.mjs` — only if a real bug in the script surfaces during the run.

### Step C.1: Run the detector end-to-end

- [ ] Run: `pnpm --filter @igrp/igrp-framework-react-design-system drift:shadcn 2>&1 | tee /tmp/drift-output.txt`

This is **slow** (network, shadcn CLI per primitive — expect 5-20 minutes). It will:
1. Init a scratch shadcn project in a temp dir.
2. For each `.tsx` under `src/components/primitives/`, run `npx shadcn@latest add <name> --dry-run --diff --base radix`.
3. Print per-file status, then a summary.

Capture the full output to `/tmp/drift-output.txt`. Exit code 0 = no drift. Exit code 1 = drift detected (expected for first run).

### Step C.2: Triage each `DRIFT` row

- [ ] For every primitive printed as `DRIFT`:
  1. Look up the file in `packages/design-system/COMPONENTS.md` "IGRP-specific deltas from upstream shadcn" section.
  2. If it has a delta entry, the drift is **expected** — disposition is "documented intentional divergence; re-stamp."
  3. If it has no delta entry, the drift is either:
     - a real upstream change we want to absorb → disposition "absorb."
     - or an undocumented IGRP divergence → either add it to the deltas section (back to Task B) or treat as "absorb."

For each "absorb" disposition, run:
```
cd <scratch-dir>  # the temp dir the script printed
npx shadcn@latest add <name> --diff <name>.tsx
```
to see the precise upstream diff, then apply the change to `packages/design-system/src/components/primitives/<name>.tsx` by hand. Re-stamp the `// shadcn: YYYY-MM-DD` header to today's date.

For each "re-stamp" disposition (intentional divergence), only update the `// shadcn:` stamp date to today's date so the next run treats this as the new baseline.

### Step C.3: Write the baseline report

- [ ] Create `packages/design-system/scripts/drift-baseline-2026-05.md`:

```markdown

Snapshot from running `pnpm drift:shadcn` against the live shadcn registry on YYYY-MM-DD. This is the first end-to-end run since the detector was introduced (`0.1.0-beta.131`).

## Summary

- Primitives audited: [COUNT — copy from script output]
- Drift detected: [COUNT_DRIFTED]
- Disposition: [N] documented intentional divergence (re-stamped); [N] upstream change absorbed; [N] deferred (see below).

| Primitive | Stamp before | Stamp after | Disposition | Notes |
| --- | --- | --- | --- | --- |
| [name.tsx] | [old date or `<none>`] | [today] | re-stamped (intentional) / absorbed / deferred | [one-line reason] |
| ... | ... | ... | ... | ... |

## Deferred entries (if any)

For each "deferred" disposition above, document why. Examples:
- "Upstream migrated to `base-ui` instead of `radix`; absorbing would require base swap — out of scope."
- "Upstream change conflicts with the Accordion delta documented in COMPONENTS.md; needs design discussion."

## Next run

The next `pnpm drift:shadcn` invocation should be quiet (zero `DRIFT` rows) for everything in the "re-stamped" and "absorbed" categories. Anything new is real drift to investigate.
```

Fill in the table from the triage results in C.2. Be terse — one line per primitive.
