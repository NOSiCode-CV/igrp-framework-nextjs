# R2 P0 — Critical Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the two P0-Critical findings from the 2026-06-29 R2 deep review, both in `@igrp/igrp-framework-react-design-system` and both shipping to every consumer app: (1) `IGRPDataTable` rebuilds its column model, column helper, and filter descriptors on every render while the React Compiler is explicitly disabled (`"use no memo"`), so TanStack resets its column caches on every unrelated parent re-render; (2) `IGRPDataTableButtonModal` hosts arbitrary content (typically a whole form) inside `<DialogDescription asChild>`, which both mis-announces the entire subtree as the dialog's `aria-describedby` description and throws `React.Children.only` for any multi-root `render`.

**Architecture:** Surgical, source-only fixes inside the Horizon data-table family. Item 1 wraps the per-render derivations (`allColumns`, `filterDescriptors`, the column helper) in `useMemo` keyed on their real inputs — the React Compiler stays off, so the caller owns `data`/`columns` stability and the component owns the derivations downstream of them. Item 2 drops `asChild` on the modal body `DialogDescription`, renders the content in a plain `<div>`, and (optionally) keeps a separate short hidden `DialogDescription` so Radix's `aria-describedby` still resolves without swallowing the form. `DialogTitle` is already correct and stays. Both are verified through the design-system Storybook package — there is **no** package-local headless unit suite for the DS.

**Tech Stack:** TypeScript 5.9, React 19.2 (`react-dom` 19.2, React Compiler via `babel-plugin-react-compiler` — opted out per-file here with `"use no memo"`), TanStack Table v8 (`@tanstack/react-table`), Radix Dialog (`@radix-ui/react-dialog`), Tailwind v4, SWC+Babel build pipeline. Verification: Storybook 10.4.6 with `@storybook/addon-vitest` (Vitest 4.1.9 browser mode via Playwright/Chromium) for `play`/interaction + a11y assertions, and `@storybook/test-runner` (Playwright) for HTML snapshots. `storybook/test` exports `within`, `userEvent`, `expect`, `waitFor`, `fn`. Node ≥ 22, pnpm workspaces, changesets (pre-release `beta`, always `patch`).

The caller still owns `data`/`columns` identity; this task documents that and stabilizes everything the component derives from them.

  parameters: { layout: 'centered' },

const schema = z.object({ name: z.string().min(1) });

/** A full form inside the modal — the canonical real-world body. */
export const WithFormBody: Story = {
  args: {
    labelTrigger: 'Edit',
    modalTitle: 'Editar utilizador',
    icon: 'Pencil',
    render: () => (
      <IGRPForm schema={schema} defaultValues={{ name: '' }} onSubmit={() => {}} className="px-6 py-4">

    expect(body).not.toBeNull();
    expect(body?.querySelector('input')).not.toBeNull();
