# Migration Guide — Template resync (`12-template-resync`)

This is a **resync migration**. Between migrations `01`–`11` and this one, several files
in `templates/demo-legacy` were edited directly without an accompanying migration.
As a result those changes shipped only to apps scaffolded from the template zip — apps
upgraded with `igrp-migrate` never received them.

This migration re-captures the current state of every drifted file so existing
consumers converge on the same template that new consumers get. It is generated to
match `@igrp/framework-next@0.1.0-beta.144` and its sibling packages.

> Going forward, the `check:drift` gate (run automatically by the migrator's `release`
> script) prevents this divergence from recurring: a template edit that isn't captured
> in a migration now fails the release.

## What changed

### Files re-captured (`file.write`, full replace)

| File | Area |
|---|---|
| `src/actions/igrp/layout.ts` | server action — layout/session fetch |
| `src/lib/config/get-session-args.ts` | session refetch config |
| `src/lib/config/get-routes.ts` | route config |
| `src/lib/config/get-base-path.ts` | basePath resolution |
| `src/lib/auth.ts` | `withIGRPAuth` instance, server session helpers |
| `src/lib/dal.ts` | data-access layer |
| `src/lib/report-error.ts` | error reporting helper |
| `src/app/api/auth/[...nextauth]/route.ts` | NextAuth route handler |
| `src/app/error.tsx` | root error boundary |
| `src/app/(auth)/error.tsx` | auth route-group error boundary |
| `src/app/(igrp)/error.tsx` | igrp route-group error boundary |
| `src/app/(auth)/logout/page.tsx` | logout flow |
| `.env.example` | documented environment variables |

### Files removed (`file.delete`)

| File | Reason |
|---|---|
| `src/app/[...not-found]/page.tsx` | The catch-all not-found route was removed from the template; global 404 handling is covered by `src/app/not-found.tsx` alone. Migration `01` shipped this file; this step deletes it from apps that applied `01`. |

### Dependency bumps (`deps.bump`)

Pins the `@igrp/*` framework packages to the versions paired with this template
snapshot (framework `0.1.0-beta.144`), and aligns the React/Next.js runtime
(`next ^15.5.18`, `react`/`react-dom 19.2.6`) — these had advanced in the template
since migration `04` without an intervening migration.

## Notes

- All replacements are full-file overwrites of the current template state — review the
  diff in your app before committing if you have local customizations to these files.
- The `file.delete` is a no-op for apps that never applied migration `01` or already
  removed the catch-all route.
- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
