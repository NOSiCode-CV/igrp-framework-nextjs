# Migration Guide — Deferred logout & cleanup (`14-deferred-logout-and-cleanup`)

This migration captures all template changes since `13-resync-beta145` and the
accompanying `@igrp/framework-next-auth@0.1.0-beta.139` release.

## What changed

### Deferred logout flow (Option A)

The logout page no longer calls `signOut()` before posting to the IdP
`end_session_endpoint`. Instead it sets a `logout_pending` cookie marker and leaves.
The IdP's redirect back to `/login` is the confirmation that the SSO session was
terminated; `LogoutCompletion` then runs the local `signOut()` at that point.

A middleware backstop completes any abandoned logout on the next protected-route
request (cookie present but IdP never returned).

| File | Change |
|---|---|
| `src/lib/logout-pending.ts` | **New file** — shared cookie helpers (`markLogoutPending`, `clearLogoutPending`) and `LOGOUT_PENDING_COOKIE` constant used by the logout page, `/login`, and middleware |
| `src/app/(auth)/login/logout-completion.tsx` | **New file** — `LogoutCompletion` client component: runs deferred `signOut()`, clears the marker, reloads `/login` |
| `src/app/(auth)/login/page.tsx` | Renders `LogoutCompletion` instead of the login form when the marker cookie is present |
| `src/app/(auth)/logout/page.tsx` | Sets marker before IdP POST; defers local `signOut()` to `/login`; removed all debug trace `console.log` calls |
| `src/middleware.ts` | Added logout backstop: forces `/login` on any protected route when `logout_pending` is set |

### Auth & OIDC cleanup (`@igrp/framework-next-auth@0.1.0-beta.139`)

Removed debug `console.debug` and `console.log` trace calls from the OIDC token
refresh path, the `events.signOut` handler, and the server action that builds the
IdP end-session URL. All `console.warn`/`console.error` calls with diagnostic value
are retained.

| File | Change |
|---|---|
| `src/actions/igrp/auth.ts` | Removed debug trace logs (function-entry, token shape, end-session URL); kept `warn` on degraded paths |
| `src/lib/auth.ts` | Removed `console.debug` on unauthenticated requests (noisy, fires on every public route) |

### Layout hardening

| File | Change |
|---|---|
| `src/app/(igrp)/layout.tsx` | Added `export const dynamic = "force-dynamic"` to prevent build-time prerender errors when `headers()`/`cookies()` are read inside the layout |

### Next.js config

| File | Change |
|---|---|
| `next.config.ts` | Commented out `output: "standalone"` (disabled during active development); added `browserDebugInfoInTerminal` experimental config for improved terminal debug output |

### Import-order alignment (Biome formatting)

The following files had their import blocks reordered by Biome — no logic changes:
`src/app/(auth)/error.tsx`, `src/app/(igrp)/error.tsx`, `src/app/error.tsx`,
`src/app/global-error.tsx`, `src/app/not-found.tsx`, `src/app/(igrp)/page.tsx`,
`src/lib/errors.ts`.

## Notes

- `src/lib/logout-pending.ts` and `src/app/(auth)/login/logout-completion.tsx` are new
  files — the migrator will create them even if they don't exist in your app.
- `next.config.ts` is replaced in full — re-apply any local customisations after the
  migration (e.g. restore `output: "standalone"` if your deployment requires it).
- The `dynamic = "force-dynamic"` export in `(igrp)/layout.tsx` is safe for all
  authenticated layouts that read session cookies on every request.
- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
