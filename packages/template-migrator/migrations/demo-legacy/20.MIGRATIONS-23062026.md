# Migration Guide — Sidebar toggle in the header (`20-sidebar-trigger-in-header`)

## What changed

The sidebar collapse/expand `SidebarTrigger` moved out of the sidebar's own
header and into the app header (`IGRPTemplateHeader`), matching the shadcn
dashboard layout. It is rendered only when the header data sets
`showIGRPSidebarTrigger: true`.

`@igrp/framework-next-ui` no longer renders a trigger inside
`IGRPTemplateSidebar`'s `SidebarHeader`, so without this config flag the toggle
would be missing entirely. This migration flips the template's header config so
the single header-mounted trigger appears.

The sidebar keeps `collapsible="icon"` — the trigger toggles between the
expanded panel and the collapsed icon rail (`SidebarRail` still re-expands from
the collapsed state), and the open/closed state persists in the `sidebar_state`
cookie.

### `src/igrp.template.config.ts` updated

```diff
- showIGRPSidebarTrigger: false,
+ showIGRPSidebarTrigger: true,
```

| File | Change |
|---|---|
| `src/igrp.template.config.ts` | `getHeaderData().showIGRPSidebarTrigger` → `true` |

## Notes

- Requires `@igrp/framework-next-ui` ≥ the release that moves the trigger into
  the header (`targetFrameworkVersion` above). On older `next-ui`, the sidebar
  header still renders its own trigger and enabling the header flag would show
  two triggers.
- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
