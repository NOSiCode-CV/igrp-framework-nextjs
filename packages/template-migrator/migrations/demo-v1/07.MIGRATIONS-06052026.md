# Migration Guide - Data Access Layer

This document covers the changes made on **2026-05-06**, following migration `06.MIGRATIONS-23042026.md`. Two related improvements:

1. A new Data Access Layer (`src/lib/dal.ts`) that centralises session verification and user extraction for server components — replacing ad-hoc `getSession()` calls scattered across layouts and pages.
2. A small correction to `src/lib/auth.ts`: `getSession()` now uses `isAuthBypass()` instead of the narrower `isPreviewMode()`, completing the auth-bypass unification started in migration `05`.

## Problem

After migration 06, server components and layouts had to call `getSession()` directly, duplicate the "redirect to /login if no session" guard, and cast the raw session object to extract user fields. There was no single place that owned the "is this request authenticated?" invariant, which meant bypass mode also had to be handled at each call site. Additionally, `getSession()` still called `isPreviewMode()` instead of the unified `isAuthBypass()` predicate introduced in migration 05, so `AUTH_PROVIDER=none` mode did not bypass the session fetch there consistently.

---

## Template Changes

### 1. New `src/lib/dal.ts` — Data Access Layer

Two exports:

**`verifySession()`** — memoised with React `cache()` so it runs exactly once per request regardless of how many server components call it. In bypass mode (either `IGRP_PREVIEW_MODE=true` or `AUTH_PROVIDER=none`) it returns a stub session so layouts can render without a real backend. Outside bypass mode it calls `getSession()` and redirects to `/login` if the session is absent.

```typescript
import { verifySession } from "@/lib/dal";

// In any server component or layout:
const session = await verifySession();
```

**`getAuthenticatedUser()`** — calls `verifySession()` internally and returns the minimal shape UI components need: `{ id, name, email, accessToken }`. Isolates consumers from the raw `Session` type and the double-cast required to reach `session.accessToken`.

```typescript
import { getAuthenticatedUser } from "@/lib/dal";

const user = await getAuthenticatedUser();
// user.id, user.name, user.email, user.accessToken
```

### 2. `src/lib/auth.ts` — `getSession()` uses `isAuthBypass()`

`getSession()` previously short-circuited on `isPreviewMode()` only. It now checks `isAuthBypass()` (which covers both preview mode and `AUTH_PROVIDER=none`), making it consistent with the rest of the bypass machinery from migration 05.

```diff
 export async function getSession() {
-  if (isPreviewMode()) return null;
+  if (isAuthBypass()) return null;
   return auth.getSession();
 }
```

## Public API Compatibility

| Symbol | Source | Status |
|---|---|---|
| `verifySession` | `@/lib/dal` | **New** |
| `getAuthenticatedUser` | `@/lib/dal` | **New** |
| `getSession` | `@/lib/auth` | Unchanged signature; now also bypasses for `AUTH_PROVIDER=none` |
| `serverSession` | `@/lib/auth` | Unchanged |
| `auth` | `@/lib/auth` | Unchanged |

No breaking changes. Any existing call to `getSession()` continues to work; the only behavioral difference is that `AUTH_PROVIDER=none` now returns `null` from `getSession()` the same way `IGRP_PREVIEW_MODE=true` did previously.

## Migration Steps

### 1. Pull the updated packages

```bash
pnpm install
pnpm build:framework
```

### 2. Create the DAL

Create `src/lib/dal.ts` from this migration. If your app already has protected server components that call `getSession()` + redirect, migrate them to call `verifySession()` instead — the guard and the bypass stub are now centralised there.

### 3. Update `auth.ts`

Replace the `isPreviewMode` import with `isAuthBypass` and update the call in `getSession()`. The `isAuthBypass` function is already present in `src/lib/utils.ts` from migration `05`.

### 4. (Optional) Migrate existing call sites

Any server component or layout that currently calls `getSession()` followed by `if (!session) redirect('/login')` can be simplified to a single `verifySession()` call. `getAuthenticatedUser()` further reduces boilerplate when you only need user identity fields.

## Verification

```bash
pnpm --filter @igrp/framework-next-template dev
```

| Scenario | Trigger | Expected |
|---|---|---|
| Baseline boot | default env | App renders; `verifySession()` returns real session from `getSession()` |
| Preview bypass | `IGRP_PREVIEW_MODE=true` | `verifySession()` returns stub session; no `getSession()` call; no redirect |
| Provider none | `AUTH_PROVIDER=none` | Same as preview bypass — stub session, no backend call |
| Unauthenticated | valid backend, no cookie | `verifySession()` calls `getSession()` → `null` → redirects to `/login` |
| User fields | any authenticated scenario | `getAuthenticatedUser()` returns `{ id, name, email, accessToken }` without manual casting |
