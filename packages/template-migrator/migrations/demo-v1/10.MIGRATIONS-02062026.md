# Migration Guide — Configurable session refetch + menu-role sync control

This document covers the changes made on **2026-06-02**, following migration `09.MIGRATIONS-25052026.md`. It bundles a release's worth of template-consumption changes: a configurable client session-refetch interval, a `syncRoles` control for the on-code menu push, the logout-hang fix, a sidebar-visibility correction, design-system token/theme alignment, and the removal of the dead `IGRP_M2M_SCOPE` env var.

## What changed

1. **`IGRP_SESSION_REFETCH_INTERVAL` is introduced.** The client `SessionProvider` refetch cadence is no longer hard-coded to `5 * 60`. `src/lib/config/get-session-args.ts` now reads `IGRP_SESSION_REFETCH_INTERVAL` (seconds) and falls back to `180` when the var is unset, non-numeric, or `<= 0`. This poll is the primary trigger for silent access-token refresh, so the default is tuned to sit *inside* Spring Authorization Server's default 5-minute access-token lifetime (~2 min margin).
2. **`IGRP_SYNC_ON_CODE_MENU_ROLES` is introduced.** `src/igrp.template.config.ts` forwards `syncOnCodeMenuRoles: process.env.IGRP_SYNC_ON_CODE_MENU_ROLES !== "false"` as the `syncRoles` argument of `syncApplicationMenus`. When `true` (the default), the on-code menu push also reconciles menu↔role assignments in Access Management; when `false`, only the menu entries are synced and existing role assignments are kept. The field is only consulted when `IGRP_SYNC_ON_CODE_MENUS=true` and the push actually runs (gated by `IGRP_SYNC_ACCESS=true`, `IGRP_PREVIEW_MODE=false`).
3. **`IGRP_M2M_SCOPE` is removed.** The OAuth2 `client_credentials` flow uses the AM server's default scope for the client; the optional scope override added in migration `08` was unused. The `.env.example` M2M comment block drops the `[&scope=IGRP_M2M_SCOPE]` reference.
4. **`NEXT_PUBLIC_IGRP_SETTINGS_URL` is documented** in `.env.example` — the base URL for settings when the apps-center is hosted on its own origin.
5. **Logout no longer hangs.** `src/app/(auth)/logout/page.tsx` is rewritten so the logout flow completes deterministically instead of stalling on session teardown. The page imports `getLogoutUrl` from `src/actions/igrp/auth.ts`, so that file ships too: it now implements RP-initiated logout via `buildEndSessionUrl` (the last migration to touch it — `04` — predates that function). `src/lib/utils.ts` is also refreshed, since `src/lib/config/get-session-args.ts` (above) consumes its `isPreviewMode` / auth-bypass helpers; the current version adds doc comments and the `sanitizeCallbackUrl` helper.
6. **The IGRP shell renders the sidebar.** `src/app/(igrp)/layout.tsx` drops the `showSidebar={false}` prop on `IGRPLayoutFull`, so the sidebar is shown by default.
7. **Design-system token/theme alignment.** `src/styles/globals.css` and `src/styles/themes.css` are updated to match the chart-token gap fix and theme-variant corrections shipped in the design system. Tokens remain imported before themes (see `tailwind-v4` rules); only semantic tokens are used.

## Why

- The previous hard-coded 5-minute refetch matched the token lifetime exactly, so a poll could land *after* expiry and miss the refresh window. Making it configurable — and defaulting below the token TTL — guarantees a refresh lands within the token's life.
- The on-code menu push (`09`) always reconciled role assignments with no way to opt out. `syncRoles` lets operators push menu structure without touching role bindings, which matters when AM roles are managed outside the template.
- `IGRP_M2M_SCOPE` was dead configuration — it misled operators into thinking scope was tunable from the template when the AM client already uses the registered default.

## Manual review steps

This migration replaces `.env.example`, `src/igrp.template.config.ts`, `src/lib/config/get-session-args.ts`, `src/actions/igrp/auth.ts`, `src/lib/utils.ts`, `src/app/(auth)/logout/page.tsx`, `src/app/(igrp)/layout.tsx`, `src/styles/globals.css`, and `src/styles/themes.css` wholesale. After applying:

1. **Diff your local `.env`** and:
   - Remove `IGRP_M2M_SCOPE` (no longer read).
   - Optionally add `IGRP_SESSION_REFETCH_INTERVAL` (default `180`; set below your IdP access-token TTL).
   - Optionally add `IGRP_SYNC_ON_CODE_MENU_ROLES` (default `true`; set `false` to keep AM role assignments untouched during the on-code menu push).
   - Optionally add `NEXT_PUBLIC_IGRP_SETTINGS_URL` if your apps-center settings live on a separate origin.
2. **If you customized `src/styles/globals.css` or `src/styles/themes.css`**, re-apply your theme overrides on top of the new files — the wholesale replace will clobber local edits. Keep the tokens import before the themes import.
3. **If you intentionally hid the sidebar** via `showSidebar={false}`, re-add the prop in `src/app/(igrp)/layout.tsx` after applying.
4. **Restart the dev server.** Confirm the session poll fires at your configured interval, logout completes without hanging, and the sidebar renders.

## Caution

`IGRP_SYNC_ON_CODE_MENU_ROLES=true` is only relevant when `IGRP_SYNC_ON_CODE_MENUS=true`, which is itself **destructive on the AM side** (it reconciles AM to match `IGRP_DEFAULT_MENU`). With role sync also on, AM menu↔role assignments are overwritten too. Only enable both in environments where the local menu array and its role bindings are authoritative.
