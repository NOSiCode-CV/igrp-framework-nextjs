# Migration Guide — Email scope + framework beta.153 (`18-email-scope-enable`)

## What changed

### OIDC `email` scope enabled (`IGRP_AUTH_SCOPES`)

The IGRP Authorization Server now advertises `scopes_supported: ["openid", "email"]`
and the OAuth client registration allows the `email` scope. The default scope string
in `.env.example` is updated from `openid` to `openid email` so the IdP returns the
`email` claim in the OIDC token.

**Action required:** Add `email` to `IGRP_AUTH_SCOPES` in your `.env`:

```diff
- IGRP_AUTH_SCOPES=openid
+ IGRP_AUTH_SCOPES=openid email
```

> If you are not ready to request the `email` scope (e.g. the `email` scope is not
> registered on your OAuth client), keep `IGRP_AUTH_SCOPES=openid`. The framework
> always ensures `openid` is included regardless of this value.

### Framework improvements (no template code changes required)

- **`next-auth`**: diagnostic `console.error` messages added when the refresh token is
  reported inactive by the introspection endpoint, or when `refreshOidcAccessToken`
  returns an error flag or throws — makes login-loop root causes visible in server logs.
- **`next`**: `use-user` switched from `unstable_cache` to `React.cache()` — prevents
  stale 401 revalidations caused by rotating access tokens being embedded in the
  `unstable_cache` key.

- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
- The `email` claim is mapped to `user.email` in the NextAuth `profile()` callback
  and surfaced via `session.user.email` — no additional wiring needed.
