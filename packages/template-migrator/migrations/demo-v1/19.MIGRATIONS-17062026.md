# Migration Guide — Adaptive session refresh (`19-adaptive-session-refresh`)

## What changed

### `IGRP_SESSION_REFETCH_INTERVAL` removed

`IGRPSessionWatcher` (shipped in `@igrp/framework-next-ui@0.1.0-beta.152`) now
schedules a `getSession()` call derived from `session.expiresAt` — exactly 45s
before the access token expires — and reschedules itself on every rotation. This
makes the old static poll interval unnecessary and removes the footgun: a
misconfigured `IGRP_SESSION_REFETCH_INTERVAL` (value ≥ access-token TTL) would
let the persisted session cookie expire between polls and bounce healthy sessions
to `/login`.

**Action required:** Remove `IGRP_SESSION_REFETCH_INTERVAL` from your `.env`
if present. The variable is no longer read.

```diff
- IGRP_SESSION_REFETCH_INTERVAL=150
```

### `src/lib/config/get-session-args.ts` updated

The helper now uses a fixed 600s backstop for `SessionProvider`'s
`refetchInterval` (last-resort only — e.g. a suspended laptop waking with a
stale session) instead of reading from the removed env var.
`refetchOnWindowFocus: true` continues to handle tab-return recovery.

| File | Change |
|---|---|
| `.env.example` | Removed `IGRP_SESSION_REFETCH_INTERVAL` |
| `src/lib/config/get-session-args.ts` | Uses 600s backstop; drops env-var read |

## Notes

- No framework dependency changes — this migration targets the same
  `0.1.0-beta.153` as migration 18.
- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
