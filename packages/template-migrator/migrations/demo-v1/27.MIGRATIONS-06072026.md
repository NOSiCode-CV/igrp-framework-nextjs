# Migration Guide — CSP + HSTS security headers (`27-csp-hsts-headers`)

## What changed

| File | Change |
|---|---|
| `src/middleware.ts` | Added `Strict-Transport-Security` (`max-age=63072000; includeSubDomains`) and a `Content-Security-Policy-Report-Only` baseline to `SECURITY_HEADERS`, alongside the existing headers. Both are gated behind the same `NODE_ENV === "production"` check as everything else in `withSecurityHeaders`. |

## Why

- `Strict-Transport-Security` and CSP were never present in this template at any point in its history — not a regression, a standing gap closed here.
- The CSP ships in **report-only** mode: it can never block a request, browsers only log violations. This is intentional — enforcing mode is a follow-up once real violation data is collected.
- `style-src 'self' 'unsafe-inline'` is a deliberate, permanent allowance: Radix (used throughout the design system's floating-UI components — Popover, DropdownMenu, Tooltip, Select, HoverCard, ContextMenu) sets inline `style` attributes at runtime with no CSP nonce escape hatch.
- `script-src 'self'` intentionally omits `'unsafe-inline'`. This **will** surface a real, expected violation from `next-themes`' inline theme-boot script (no nonce wired yet). Closing that gap for real requires threading a `nonce` through `IGRPThemeProvider` into `next-themes`' `nonce` prop — a `framework-next-ui` change, tracked separately, not part of this migration.
- `img-src`/`form-action` are built dynamically in the file from `NEXT_PUBLIC_ALLOWED_DOMAINS` and `IGRP_AUTH_ISSUER` respectively, degrading gracefully (falling back to just `'self'`) when either is unset or malformed — no new required env vars.

## Notes

- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
- No dependency bump, no new env vars — this migration only touches `src/middleware.ts`.
