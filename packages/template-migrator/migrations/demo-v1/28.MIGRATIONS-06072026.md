# Migration Guide — Resync to beta.165 + query-provider split (`28-resync-beta165-query-provider-split`)

## What changed

Closes out drift accumulated since migrations 25/26, most of it pre-existing
(unlogged edits to core auth/layout/error files) plus one architectural
change to the React Query provider setup.

### File changes

| File | Change |
|---|---|
| `src/lib/auth.ts` | `serverSession()` no longer seeds the access-client config with a token from a session carrying `session.error` (refresh already failed) — avoids seeding a guaranteed-dead token. |
| `src/app/(igrp)/layout.tsx`, `src/igrp.template.config.ts`, `src/lib/config/get-pkj.ts`, `src/app/(auth)/login/page.tsx`, `src/lib/dal.ts`, `src/app/not-found.tsx`, `src/actions/igrp/auth.ts`, `src/app/global-error.tsx`, `src/app/error.tsx`, `src/app/(auth)/error.tsx`, `src/app/(igrp)/error.tsx`, `src/app/(auth)/logout/page.tsx`, `src/app/(igrp)/page.tsx`, `src/lib/errors.ts`, `src/app/(auth)/login/logout-completion.tsx`, `src/app/layout.tsx` | Resync — accumulated drift from unlogged template edits since migrations 25/26. |

### Query provider split

`src/providers/query-client.tsx` is removed and replaced by three files:
- `src/providers/query-client.ts` — shared `makeQueryClient()` factory (client + server defaults in one place).
- `src/providers/query-client.server.ts` — `"server-only"`, `React.cache()`-memoized per-request server QueryClient built from the same factory.
- `src/providers/query-provider.tsx` — client `QueryClientProvider`, reusing the same factory so SSR-prefetched queries hydrate with matching `staleTime`/`gcTime`/`retry` behavior.

### Dependency bumps (`deps.bump`)

| Dependency | Version |
|---|---|
| `@igrp/framework-next` | `0.1.0-beta.165` |
| `@igrp/framework-next-ui` | `0.1.0-beta.161` |

`@igrp/framework-next-auth`, `@igrp/framework-next-types`, and `@igrp/igrp-framework-react-design-system` are unchanged from migration 25/26's pins — still current.

## Notes

- Run `igrp-migrate plan` first to preview, then `igrp-migrate apply`.
- Not covered by this migration, and deliberately so: `(demo)` showcase route-group pages (`src/app/(igrp)/(demo)/**`), `src/temp/**` preview-mode mock data, `.gitkeep` placeholders, `package.json`, and `biome.json`. These exist in the template but are either example/scaffold content not meant to be force-pushed into an already-customized consumer app, or (for `package.json`) already covered narrowly by `deps.bump` instead of a full-file replace to avoid clobbering a consumer's own scripts/deps.
