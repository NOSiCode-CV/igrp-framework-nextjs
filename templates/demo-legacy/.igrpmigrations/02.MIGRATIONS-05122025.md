## Access Management Synchronization

### Overview
Implemented automatic synchronization of application metadata, routes, and menus with the IGRP Access Management API. This allows applications to automatically register their routes and menus in the access management system during startup.

### Changes Made

#### 1. Route Synchronization (`packages/framework/next/src/lib/sync-routes.ts`)

**Created:**
- New function `igrpSyncRoutes()` that synchronizes static routes to Access Management
- Extracts routes from Next.js generated type definitions
- Filters out dynamic routes and excluded routes (login, logout, catch-all)
- Creates resource items for each route

**Key Features:**
```typescript
export async function igrpSyncRoutes({
  baseUrl,
  m2mServiceId,
  m2mToken,
  appRoutes,
  paramMapBody,
}: IGRPSyncRoutesArgs)
```

**Functionality:**
- Parses `ParamMap` interface from Next.js routes type definitions
- Filters static routes (routes without parameters)
- Excludes system routes (`/login`, `/logout`, `/[...not-found]`)
- Creates `ResourceDTO` with route items
- Syncs to Access Management API using M2M authentication

**Benefits:**
- Automatic route registration in access management
- No manual route configuration needed
- Routes are available for menu assignment
- Supports resource-based access control

---

#### 2. Menu Synchronization (`packages/framework/next/src/lib/sync-menus.ts`)

**Created:**
- New function `igrpSyncMenus()` that synchronizes menu items to Access Management
- Maps IGRP menu types to Access Management API types
- Supports conditional synchronization via environment variable

**Key Features:**
```typescript
export async function igrpSyncMenus({
  appCode,
  menus,
  baseUrl,
  m2mServiceId,
  m2mToken,
  syncEnabled,
}: IGRPSyncMenusArgs)
```

**Functionality:**
- Checks if menu synchronization is enabled (`IGRP_SYNC_ON_CODE_MENUS`)
- Maps menu items from IGRP format to Access Management format
- Syncs menus to specific application code
- Provides clear logging for sync status

**Benefits:**
- Automatic menu synchronization from code to access management
- Supports code-first menu definition
- Can be disabled for production environments
- Maintains menu structure and hierarchy

---

#### 3. Application Synchronization (`packages/framework/next/src/lib/sync-application.ts`)

**Created:**
- New function `igrpSyncApplication()` that synchronizes application metadata
- Extracts application information from package.json
- Creates or updates application in Access Management

**Key Features:**
```typescript
export async function igrpSyncApplication({
  appInformation,
  baseUrl,
  appCode,
  m2mServiceId,
  m2mToken,
}: IGRPSyncApplicationArgs)
```

**Functionality:**
- Creates `ApplicationDTO` from package.json information
- Generates application code if not provided
- Sets application type as `INTERNAL`
- Sets status as `ACTIVE`
- Syncs to Access Management API

**Benefits:**
- Automatic application registration
- Metadata synchronization (name, description, version)
- Consistent application codes across environments
- Supports application discovery

---

#### 4. Startup Sync Orchestration (`packages/framework/next/src/lib/startup-sync.ts`)

**Created:**
- New function `igrpStartupSync()` that orchestrates all synchronization operations
- Ensures sync only runs once per application instance
- Provides comprehensive error handling

**Key Features:**
```typescript
export async function igrpStartupSync({
  syncEnabled,
  appInformation,
  baseUrl,
  appCode,
  m2mServiceId,
  m2mToken,
  menus,
  appRoutes,
  paramMapBody,
}: IGRPStartupSyncArgs)
```

**Functionality:**
- Checks if synchronization is enabled (`IGRP_SYNC_ACCESS`)
- Uses singleton pattern to prevent duplicate syncs
- Executes syncs in order: Application → Routes → Menus
- Provides detailed logging for debugging
- Handles errors gracefully without breaking application startup

**Sync Order:**
1. Application metadata sync
2. Routes sync
3. Menus sync

**Benefits:**
- Single entry point for all sync operations
- Prevents duplicate syncs
- Comprehensive error handling
- Detailed logging for troubleshooting

---

#### 5. Integration in Root Layout (`packages/framework/next/src/layouts/igrp-root-layout.tsx`)

**Modified:**
- Added call to `igrpStartupSync()` during layout initialization
- Extracts sync configuration from `apiManagementConfig`
- Passes sidebar menu items for synchronization

**Key Changes:**
```typescript
await igrpStartupSync({
  syncEnabled: syncAccess,
  appInformation,
  baseUrl: apiManagementConfig?.baseUrl || '',
  appCode,
  menus: sidebarData.menuItems || [],
  m2mServiceId: apiManagementConfig?.m2mServiceId || '',
  m2mToken: apiManagementConfig?.m2mToken || '',
  appRoutes: apiManagementConfig?.appRoutes,
  paramMapBody: apiManagementConfig?.paramMapBody,
});
```

**Benefits:**
- Automatic sync on application startup
- Uses configuration from template config
- Accesses sidebar data for menu sync
- Runs before page rendering

## Configuration Refactoring

### Overview
Refactored `igrp.template.config.ts` to use modular helper functions, improving maintainability and testability.

### Changes Made

#### 1. Template Configuration (`src/igrp.template.config.ts`)

**Before:**
- Inline configuration logic
- Direct environment variable access
- Hardcoded session arguments
- No route extraction

**After:**
- Uses helper functions from `lib/config/`
- Modular configuration structure
- Extracted route information
- Added `apiManagementConfig` section

**Key Changes:**
```typescript
import { getPackageJson } from "./lib/config/get-pkj";
import { getSessionArgs } from "./lib/config/get-session-args";
import { getRoutes } from "./lib/config/get-routes";

// ...

const routes = getRoutes();
const appRoutes = routes?.appRoutes ?? [];
const paramMapBody = routes?.paramMapBody ?? "";

// ...

apiManagementConfig: {
  baseUrl: process.env.IGRP_ACCESS_MANAGEMENT_API || "",
  m2mServiceId: process.env.IGRP_M2M_SERVICE_ID || "",
  m2mToken: process.env.IGRP_M2M_TOKEN || "",
  syncOnCodeMenus: process.env.IGRP_SYNC_ON_CODE_MENUS === "true",
  appRoutes,
  paramMapBody,
},
sessionArgs: getSessionArgs(),
```

**Benefits:**
- Improved code organization
- Easier to test individual functions
- Better separation of concerns
- Reusable configuration helpers

### 2. Session Arguments Helper (`src/lib/config/get-session-args.ts`)

**Created:**
- Centralizes session configuration logic
- Handles preview mode detection
- Returns appropriate session arguments based on mode

**Key Features:**
```typescript
export function getSessionArgs() {
  if (isPreviewMode()) {
    return {
      refetchInterval: 0,
      refetchOnWindowFocus: false,
      basePath: getBasePath(process.env.NEXT_PUBLIC_BASE_PATH || ""),
    };
  }

  return {
    refetchInterval: 5 * 60,
    refetchOnWindowFocus: true,
    basePath: getBasePath(process.env.NEXT_PUBLIC_BASE_PATH || ""),
  };
}
```

**Functionality:**
- Checks preview mode status
- Disables session refetching in preview mode
- Enables normal session polling in production
- Normalizes base path for authentication

**Benefits:**
- Consistent session configuration
- Preview mode support
- Centralized base path handling

### 3. Routes Extraction Helper (`src/lib/config/get-routes.ts`)

**Created:**
- Extracts route information from Next.js generated type definitions
- Parses `AppRoutes` type and `ParamMap` interface
- Returns routes array and parameter map body

**Key Features:**
```typescript
export function getRoutes() { 
  try {
    const file = path.join(process.cwd(), ".next/types/routes.d.ts");
    const content = fs.readFileSync(file, "utf8");

    const appRoutesMatch = content.match(/type AppRoutes\s*=\s*([\s\S]*?)\n(?=type|interface)/);
    const paramMapMatch = content.match(/interface ParamMap\s*{([\s\S]*?)^}/m);

    // ... parsing logic ...

    return { appRoutes, paramMapBody };
  } catch (error) {
    console.warn("Could not read routes file:", error);
  }  
}
```

**Functionality:**
- Reads Next.js generated routes type file
- Extracts `AppRoutes` union type
- Extracts `ParamMap` interface body
- Parses routes and filters empty values
- Handles errors gracefully

**Benefits:**
- Automatic route discovery
- No manual route configuration
- Supports dynamic route detection
- Works with Next.js App Router

### 4. Base Path Helper (`src/lib/config/get-base-path.ts`)

**Created:**
- Normalizes base path for authentication endpoints
- Handles edge cases (leading/trailing slashes)
- Provides default path if not specified

**Key Features:**
```typescript
export function getBasePath(bp: string) {
  if (!bp) return "/api/auth";

  const normalized = bp.startsWith("/") ? bp : `/${bp}`;
  const cleanPath = normalized.endsWith("/")
    ? normalized.slice(0, -1)
    : normalized;
  return `${cleanPath}/api/auth`;
}
```

**Functionality:**
- Returns default `/api/auth` if base path is empty
- Ensures leading slash is present
- Removes trailing slash
- Appends `/api/auth` to base path

**Benefits:**
- Consistent base path formatting
- Handles various input formats
- Prevents path-related errors

## Environment Variables

### New Environment Variables

#### Access Management Configuration

**`IGRP_ACCESS_MANAGEMENT_API`**
- **Description:** Base URL for the Access Management API
- **Example:** `https://api.igrp.cv/access-management`
- **Required:** Yes (if sync is enabled)

**`IGRP_M2M_SERVICE_ID`**
- **Description:** Machine-to-machine service identifier for API authentication
- **Example:** `my-app-service`
- **Required:** Yes (if sync is enabled)

**`IGRP_M2M_TOKEN`**
- **Description:** Machine-to-machine authentication token
- **Example:** `eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...`
- **Required:** Yes (if sync is enabled)

**`IGRP_SYNC_ACCESS`**
- **Description:** Enable/disable Access Management synchronization
- **Values:** `true` or `false`
- **Default:** `false`
- **Required:** No

**`IGRP_SYNC_ON_CODE_MENUS`**
- **Description:** Enable/disable menu synchronization from code
- **Values:** `true` or `false`
- **Default:** `false`
- **Required:** No
- **Note:** Only works when `IGRP_SYNC_ACCESS=true`

## Migration Steps

### To Enable Access Management Synchronization

1. **Set Required Environment Variables:**
   ```bash
   IGRP_ACCESS_MANAGEMENT_API=https://api.igrp.cv/access-management
   IGRP_M2M_SERVICE_ID=your-service-id
   IGRP_M2M_TOKEN=your-m2m-token
   IGRP_SYNC_ACCESS=true
   ```

2. **Optional: Enable Menu Synchronization:**
   ```bash
   IGRP_SYNC_ON_CODE_MENUS=true
   ```

3. **Restart Development Server:**
   ```bash
   pnpm dev
   ```

4. **Verify Synchronization:**
   - Check console logs for sync status
   - Verify application appears in Access Management
   - Check routes are registered
   - Verify menus are synced (if enabled)

### To Disable Synchronization

1. **Set Environment Variable:**
   ```bash
   IGRP_SYNC_ACCESS=false
   ```
   Or remove the variable entirely

2. **Restart Development Server**

3. **Verify:**
   - No sync logs in console
   - Application runs normally without sync

## Notes

- Synchronization runs once per application instance (singleton pattern)
- Routes are extracted from Next.js generated type definitions (`.next/types/routes.d.ts`)
- Only static routes (without parameters) are synchronized
- System routes (`/login`, `/logout`, `/[...not-found]`) are excluded
- Menu synchronization can be enabled independently via `IGRP_SYNC_ON_CODE_MENUS`
- All sync operations use M2M (machine-to-machine) authentication
- Errors during sync do not prevent application startup
- Preview mode disables session refetching automatically
