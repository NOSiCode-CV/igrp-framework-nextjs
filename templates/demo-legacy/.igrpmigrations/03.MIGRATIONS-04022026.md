# Migration Guide - Tailwind v4 source scanning & token-only imports

This document explains the migration applied to the demo template styling setup to avoid CSS conflicts and ensure Tailwind utilities are generated consistently.

## Overview

The demo template now compiles Tailwind **once** (in the app) and relies on Tailwind v4 `@source` to scan:

- The demo app source files
- The compiled `dist/` output of IGRP packages used by the app

This removes the need to import prebuilt `@igrp/*/styles.css` files (which can:
miss utilities and/or override each other depending on import order).

---

## `globals.css` changes

File: `src/styles/globals.css`

### Before (problematic approach)

- Imported prebuilt package CSS:
  - `@igrp/igrp-framework-react-design-system/styles.css`
  - `@igrp/framework-next-ui/styles.css`
- Result:
  - CSS cascade/import order conflicts
  - Missing Tailwind utilities (because prebuilt CSS is static)

### After (recommended approach)

- Use Tailwind v4 `@source` so the app generates all needed utilities:

```css
@source "../**/*.{ts,tsx,js,jsx}";
@source "../../node_modules/@igrp/framework-next-ui/dist/**/*.{js,jsx,ts,tsx,mjs,cjs}";
@source "../../node_modules/@igrp/igrp-framework-react-design-system/dist/**/*.{js,jsx,ts,tsx,mjs,cjs}";
```

- Import **tokens only** (CSS variables), then theme overrides:

```css
@import "@igrp/igrp-framework-react-design-system/tokens";
@import "./themes.css";
```

- Do **not** import `@igrp/*/styles.css` from the packages in the demo.

---

## Why this change

- **Avoid conflicts**: importing multiple prebuilt `styles.css` files can lead to unexpected overrides.
- **No missing utilities**: Tailwind v4 generates utilities based on what it scans; prebuilt CSS may not include utilities your pages need.
- **One token source of truth**: the demo uses the design-system tokens and overrides them via `themes.css`.

---

## Migration steps for existing apps

1. Update `src/styles/globals.css`:
   - Remove any imports of:
     - `@igrp/igrp-framework-react-design-system/styles.css`
     - `@igrp/framework-next-ui/styles.css`
   - Add `@source` entries for:
     - Your app sources
     - The `dist/` output of the IGRP packages you use
   - Import `@igrp/igrp-framework-react-design-system/tokens`
   - Keep your theme overrides (like `./themes.css`) **after** tokens

2. Restart the dev server so Tailwind picks up the new `@source` scanning:

```bash
pnpm dev
```

---

## Notes

- If you add/remove IGRP packages that contain Tailwind classes, update the `@source` paths accordingly.
- Theme customization should be done by overriding CSS variables (tokens) in `src/styles/themes.css`.
