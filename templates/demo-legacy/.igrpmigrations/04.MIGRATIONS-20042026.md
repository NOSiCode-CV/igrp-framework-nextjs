## Multi Auth Provider Support

### Overview

The authentication layer was extended to support multiple identity providers (Keycloak, Autentika/WSO2IS, and `none`) selected via the `AUTH_PROVIDER` environment variable. Previously, Keycloak credentials were hardcoded throughout the codebase. All provider-specific logic is now delegated to `@igrp/framework-next-auth` helpers.

### Changes Made

#### 1. Login Page (`src/app/(auth)/login/page.tsx`)

**Before:**
- No provider awareness — always rendered a generic login form.

**After:**
- Reads the active provider via `getAuthProviderIdFromEnv()` and passes it to `IGRPAuthForm` so the form can adapt its UI per provider.

```typescript
import { getAuthProviderIdFromEnv } from "@igrp/framework-next-auth";

const providerId = getAuthProviderIdFromEnv(process.env);

<IGRPAuthForm
  ...
  providerId={providerId}
/>
```

#### 2. `.env.example`

**Added:**

```env
# Active provider
# Supported values: keycloak | autentika | none
AUTH_PROVIDER=keycloak

# Autentika Client ID
AUTENTIKA_CLIENT_ID=

# Autentika Client Secret
AUTENTIKA_CLIENT_SECRET=

# Autentika Host
# Example: https://autentika.example.com
AUTENTIKA_HOST=

# Autentika Tenant Name
# Example: carbon.super
AUTENTIKA_TENANT_NAME=carbon.super

# Autentika scopes
# Example: openid internal_login
AUTENTIKA_SCOPES=openid internal_login
```

## Auth Module Consolidation

### Overview

The auth layer was refactored across two commits to consolidate boilerplate into a single central `src/lib/auth.ts` file using `withIGRPAuth()` from `@igrp/framework-next-auth/config`. The intermediate files `auth-helpers.ts`, `auth-options.ts`, and `types/next-auth.d.ts` were deleted.

### Files Deleted

- `src/lib/auth-helpers.ts` — token helpers (getAccessToken, refreshAccessToken)
- `src/lib/auth-options.ts` — NextAuth `authOptions` object
- `types/next-auth.d.ts` — manual session/JWT type augmentation

### Files Created

#### `src/lib/auth.ts` (new central auth module)

```typescript
import { withIGRPAuth } from "@igrp/framework-next-auth/config";
import { redirect } from "next/navigation";
import { igrpSetAccessClientConfig } from "@igrp/framework-next";
import { assertAuthProviderEnv } from "@igrp/framework-next-auth";
import { isPreviewMode } from "@/lib/utils";

export const auth = withIGRPAuth({
  onSessionExpired: () => redirect("/logout"),
});

export async function serverSession() {
  const apiManagement = process.env.IGRP_ACCESS_MANAGEMENT_API || "";
  try {
    if (!process.env.NEXTAUTH_SECRET) {
      console.warn("NEXTAUTH_SECRET is not set. This is required for production.");
      if (process.env.NODE_ENV === "production") {
        throw new Error("NEXTAUTH_SECRET must be set in production");
      }
    }
    assertAuthProviderEnv(process.env);
    const session = await auth.serverSession();
    if (session !== null) {
      igrpSetAccessClientConfig({
        token: session.accessToken as string,
        baseUrl: apiManagement,
      });
    }
    return session;
  } catch (error) {
    console.error("::Error getting server session::", error);
    return null;
  }
}

export async function getSession() {
  if (isPreviewMode()) return null;
  return auth.getSession();
}
```

**Benefits:**
- Single source of truth for auth configuration
- Provider is resolved automatically from `AUTH_PROVIDER` env var
- Session type augmentation is handled internally by `withIGRPAuth`
- All consumers (`middleware`, `layout`, `route handler`) import from `@/lib/auth`

#### `src/actions/igrp/auth.ts` (simplified to re-export)

**Before:** contained full `serverSession()` and `getSession()` implementations.

**After:** delegates entirely to `src/lib/auth`:

```typescript
"use server";

export { serverSession, getSession } from "@/lib/auth";
```

#### `src/app/api/auth/[...nextauth]/route.ts`

**Before:**
```typescript
import NextAuth from "next-auth";
import { authOptions } from "@/lib/auth-options";
const handler = NextAuth({ ...authOptions, debug: process.env.NODE_ENV === "development" });
export { handler as GET, handler as POST };
```

**After:**
```typescript
import { auth } from "@/lib/auth";
export const { GET, POST } = auth;
```

#### `src/actions/igrp/layout.ts`

**Before:**
```typescript
import { getAccessToken } from "@/lib/auth-helpers";
// ...
: await getAccessToken();
```

**After:**
```typescript
import { auth } from "@/lib/auth";
// ...
: await auth.getAccessToken();
```

## Middleware Hardening

### Overview

The middleware was rewritten to delegate all auth logic to the central `auth` instance and adds security headers to all responses in production.

### Changes Made (`src/middleware.ts`)

**Before:**
- Used raw `getToken()` and `isAuthDisabled()` from `@igrp/framework-next-auth`
- No security headers
- Token expiry check was inlined
- Static `config` object was hardcoded in the file

**After:**
- Delegates to `auth.isAuthDisabled()`, `auth.isPreviewMode()`, `auth.getTokenFromRequest()`, `auth.isTokenExpiredOrFailed()`, `auth.getLoginRedirectUrl()`
- Adds production security headers (`X-Content-Type-Options`, `X-Frame-Options`, `X-XSS-Protection`, `Referrer-Policy`, `Permissions-Policy`) via `withSecurityHeaders()`
- Exports `config` from `auth` (dynamic matcher managed by the framework)

```typescript
import { auth } from "@/lib/auth";

export async function middleware(request: NextRequest) {
  const { pathname } = request.nextUrl;

  if (auth.isAuthDisabled()) return withSecurityHeaders(NextResponse.next());
  if (auth.isPreviewMode()) return withSecurityHeaders(NextResponse.next());
  if (isPublicPath(pathname)) return withSecurityHeaders(NextResponse.next());

  const loginRedirect = () =>
    withSecurityHeaders(NextResponse.redirect(auth.getLoginRedirectUrl(request)));

  let token: Awaited<ReturnType<typeof auth.getTokenFromRequest>>;
  try {
    token = await auth.getTokenFromRequest(request);
  } catch (error) {
    console.error("[Middleware] getToken failed", error, { pathname });
    return loginRedirect();
  }

  if (!token || auth.isTokenExpiredOrFailed(token)) {
    return loginRedirect();
  }

  return withSecurityHeaders(NextResponse.next());
}

export const { config } = auth;
```

## Environment Variables

### New Variables

| Variable | Description | Required |
|---|---|---|
| `AUTH_PROVIDER` | Active auth provider: `keycloak`, `autentika`, or `none` | Yes |
| `AUTENTIKA_CLIENT_ID` | Client ID registered in Autentika/WSO2IS | If using Autentika |
| `AUTENTIKA_CLIENT_SECRET` | Secret key for the Autentika client | If using Autentika |
| `AUTENTIKA_HOST` | Base URL of the Autentika/WSO2IS server | If using Autentika |
| `AUTENTIKA_TENANT_NAME` | Tenant name (default: `carbon.super`) | If using Autentika |
| `AUTENTIKA_SCOPES` | OAuth scopes (default: `openid internal_login`) | If using Autentika |

### Using `AUTH_PROVIDER=none`

Setting `AUTH_PROVIDER=none` disables authentication entirely (equivalent to preview mode but at the provider level). Useful for local development without any identity provider.

## Notes

- `withIGRPAuth()` internally handles NextAuth `authOptions`, token refresh, session type augmentation, and the route handler — no manual configuration needed.
- The `AUTH_PROVIDER=none` option disables authentication at the provider level (distinct from `IGRP_PREVIEW_MODE=true` which bypasses middleware checks).
- Security headers are only applied in `NODE_ENV=production` to avoid noise during development.
- The middleware `config` matcher is now managed by the framework via `export const { config } = auth`.
