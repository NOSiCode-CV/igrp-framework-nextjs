## Overview

The demo template has been significantly improved based on code review recommendations. Changes focus on:

- **Environment validation** at startup to fail fast when required vars are missing
- **Type safety** with proper Session types and removal of `as any`
- **Error handling** with custom error classes and consistent try/catch patterns
- **Security** via input sanitization, secure cookies, and security headers
- **Code organization** with centralized constants, helpers, and env setup

## Environment Variable Validation

### Overview

Implemented runtime validation of environment variables using Zod. In production (when not in preview mode), the app throws at startup if required vars are missing.

### Changes Made

#### 1. Environment Schema (`src/lib/env.ts`)

**Created:**

- Zod schema for all environment variables
- Optional string/URL transformers
- Preview mode enum (`"true" | "false"`)
- Cached validation result per process

**Key Features:**

```typescript
const envSchema = z.object({
  NODE_ENV: z.enum(["development", "production", "test"]).optional(),
  NEXTAUTH_SECRET: optionalString,
  NEXTAUTH_URL: optionalUrl,
  KEYCLOAK_CLIENT_ID: optionalString,
  KEYCLOAK_CLIENT_SECRET: optionalString,
  KEYCLOAK_ISSUER: optionalUrl,
  // ... IGRP and Next.js public vars
});

export function validateEnv(): Env {
  // Cached per process
  // In production (non-preview): throws if NEXTAUTH_SECRET, KEYCLOAK_* missing
  // In development or preview: parses only, no strict requirement
}
```

**Required in production (when auth is used):**

- `NEXTAUTH_SECRET`
- `KEYCLOAK_CLIENT_ID`
- `KEYCLOAK_CLIENT_SECRET`
- `KEYCLOAK_ISSUER`

---

#### 2. Environment Setup (`src/lib/env-setup.ts`)

**Created:**

- Centralized `setupEnvironment()` called once in root layout and IGRP layout
- Unsets empty env vars to avoid "Invalid URL" crashes
- Calls `validateEnv()` for schema validation

**Key Features:**

```typescript
export function setupEnvironment() {
  unsetEmptyEnv("NEXTAUTH_URL");
  unsetEmptyEnv("NEXTAUTH_URL_INTERNAL");
  validateEnv();
}
```

## Type Safety Improvements

### Overview

Removed `as any` assertions and introduced proper typing for preview sessions and layout config.

### Changes Made

#### 1. Layout Action (`src/actions/igrp/layout.ts`)

**Before:**

- `as any` for preview session object

**After:**

- `createPreviewSession()` returns typed `Session` from `@igrp/framework-next-auth`
- Uses `AUTH_CONSTANTS` for token and expiry values

**Key Changes:**

```typescript
import type { Session } from "@igrp/framework-next-auth";

function createPreviewSession(): Session {
  return {
    user: { name: "Preview User", email: "preview@example.com" },
    accessToken: AUTH_CONSTANTS.PREVIEW_TOKEN,
    expiresAt: Date.now() + AUTH_CONSTANTS.PREVIEW_SESSION_EXPIRY_MS,
    expires: "",
  };
}

const session: Session | null = isPreviewMode()
  ? createPreviewSession()
  : ((await getAccessToken()) as Session | null);
```

## Error Handling & Custom Errors

### Overview

Introduced custom error classes and consistent error handling across auth, middleware, and layout.

### Changes Made

#### 1. Custom Errors (`src/lib/errors.ts`)

**Created:**

- `EnvValidationError` – thrown when required env vars are missing
- `AuthError` – for authentication-related failures
- Re-exports `logger` for backwards compatibility

**Key Features:**

```typescript
export class EnvValidationError extends Error {
  constructor(message: string, public readonly missingVars: string[] = []) {
    super(message);
    this.name = "EnvValidationError";
  }
}

export class AuthError extends Error {
  constructor(message: string, public readonly code: string) {
    super(message);
    this.name = "AuthError";
  }
}
```

#### 2. Auth Actions (`src/actions/igrp/auth.ts`)

- `validateEnv()` called before `getServerSession`
- Catches `EnvValidationError`; in production: rethrows; in dev: returns null
- Session expiry and refresh failure handling with redirect to `/logout`

#### 3. Middleware (`src/middleware.ts`)

- Try/catch around `getToken`; on failure: logs and redirects to login

#### 4. IGRP Layout (`src/app/(igrp)/layout.tsx`)

- Try/catch around URL construction for login path; fallback to `/login` on error

## File System Operations

### Overview

Replaced synchronous file reads with async operations and added caching for serverless compatibility.

### Changes Made

#### 1. Routes Extraction (`src/lib/config/get-routes.ts`)

**Before:**

- `fs.readFileSync` (synchronous)
- `console.warn` for errors

**After:**

- `fs.promises.readFile` (async)
- `unstable_cache` with 60s revalidate for layout renders
- `logger.warn` with structured metadata

**Key Benefits:**

- Safe for serverless/edge runtimes
- Fewer file system reads during layout renders
- Consistent logging with structured metadata

## Input Validation & Security

### Overview

Added input sanitization utilities to prevent open redirects, path traversal, and XSS.

### Changes Made

#### 1. Sanitization (`src/lib/sanitize.ts`)

**Created:**

- `sanitizeString()` – trims, limits length, removes control characters
- `escapeHtml()` – escapes HTML special characters for raw HTML

**Usage:**

- `auth-options.ts` – `sanitizeRedirectUrl` in redirect callback

## Migration Steps

### For Existing Projects

1. **Ensure required files exist:**

   - `src/lib/env.ts` – env validation
   - `src/lib/env-setup.ts` – central env setup
   - `src/lib/errors.ts` – custom errors
   - `src/lib/constants.ts` – auth constants
   - `src/lib/sanitize.ts` – input sanitization
   - `src/lib/logger.ts` – structured logger
   - `src/types/env.d.ts` – env type declarations

2. **Update root layout:**

   - Import `setupEnvironment` from `@/lib/env-setup`
   - Call `setupEnvironment()` at the start of the layout (before any async logic)

3. **Update IGRP layout:**

   - Import and call `setupEnvironment()` at the start

4. **Update auth actions:**

   - Call `validateEnv()` before `getServerSession`
   - Catch `EnvValidationError` and handle appropriately

5. **Update middleware:**

   - Add try/catch around `getToken`
   - Use `AUTH_CONSTANTS.TOKEN_REFRESH_BUFFER_MS` for expiry check
   - Add security headers via `withSecurityHeaders()`

6. **Update auth-options:**

   - Use `sanitizeRedirectUrl` in redirect callback
   - Set `useSecureCookies: process.env.NODE_ENV === "production"`

7. **Restart:**

   ```bash
   pnpm dev
   ```

## Notes

- Environment validation is cached per process; subsequent calls return the same object.
- In preview mode: env validation does not require strict vars; production still enforces them.
- `setupEnvironment()` is called in both root and IGRP layouts; `validateEnv()` is cached and safe.
- Security headers are applied only in production.
- `sanitizeRedirectUrl` prevents open redirect and path traversal.
