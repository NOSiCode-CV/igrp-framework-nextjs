## Overview

The demo template has been updated with:

- **Code quality:** Single source of truth for config, env setup, and login path
- **Metadata:** `metadataBase`, `openGraph`, and `twitter` for better sharing and SEO
- **Error handling:** Try/catch in root layout and `error.tsx` boundary
- **Web Interface Guidelines:** Accessibility (skip link, ARIA, semantics), `prefers-reduced-motion`, typography (ellipsis), and anti-pattern fixes

## Remove Duplicate `setupEnvironment()`

### Overview DT

`setupEnvironment()` was called in both root layout and `(igrp)` layout, causing `validateEnv()` to run twice per request.

### Changes Made DT

**Before:**

- `src/app/layout.tsx` – called `setupEnvironment()`
- `src/app/(igrp)/layout.tsx` – called `setupEnvironment()`

**After:**

- `src/app/layout.tsx` – calls `setupEnvironment()` (unchanged)
- `src/app/(igrp)/layout.tsx` – **removed** `setupEnvironment()` and its import

### Migration Step

1. Open `src/app/(igrp)/layout.tsx`
2. Remove the import: `import { setupEnvironment } from "@/lib/env-setup";`
3. Remove the call `setupEnvironment();` at the start of the layout function

## Consolidate Config Creation

### Overview CCC

Root layout and `(igrp)` layout both called `configLayout()` and `createConfig()`. A single `getConfig()` function was introduced as the entry point.

### Changes Made CCC

#### 1. New `getConfig()` (`src/igrp.template.config.ts`)

**Added:**

```typescript
export const getConfig = cache(async (): Promise<IGRPConfigArgs> => {
  const layoutConfig = await configLayout();
  return createConfig(layoutConfig);
});
```

#### 2. Layout Updates

**`src/app/layout.tsx` – Before:**

```typescript
const [layoutConfig, { IGRPRootLayout }] = await Promise.all([
  configLayout(),
  import("@igrp/framework-next"),
]);
const config = await createConfig(layoutConfig);
```

**After:**

```typescript
const [config, { IGRPRootLayout }] = await Promise.all([
  getConfig(),
  import("@igrp/framework-next"),
]);
```

**`src/app/(igrp)/layout.tsx` – Same pattern:** replace `configLayout()` + `createConfig()` with `getConfig()`.

### Migration Steps

1. In `src/igrp.template.config.ts`:
   - Add `import { configLayout } from "@/actions/igrp/layout";`
   - Add `getConfig` as shown above (before `createConfig`)

2. In `src/app/layout.tsx`:
   - Replace `import { configLayout } from "@/actions/igrp/layout";` and `import { createConfig } from "@/igrp.template.config";` with `import { getConfig } from "@/igrp.template.config";`
   - Replace the `configLayout()` + `createConfig()` logic with `getConfig()`

3. In `src/app/(igrp)/layout.tsx`:
   - Replace `import { createConfig } from "@igrp/template-config";` and `import { configLayout } from "@/actions/igrp/layout";` with `import { getConfig } from "@/igrp.template.config";` (or `@igrp/template-config` if your tsconfig maps it)
   - Replace the `configLayout()` + `createConfig()` logic with `getConfig()`

## Simplify Layout Mock Data Functions

### Overview SLMDF

`getHeaderData` and `getSidebarData` were declared as `async () => ({...})` but used synchronous data. They were changed to sync functions returning `Promise.resolve()`.

### Changes Made

**`src/igrp.template.config.ts` – Before:**

```typescript
getHeaderData: async () => ({
  user: user,
  userProfileUrl: process.env.NEXT_PUBLIC_IGRP_PROFILE_URL || "",
  // ...
}),
getSidebarData: async () => ({
  menuItems: menu,
  user: user,
  // ...
}),
```

**After:**

```typescript
getHeaderData: () =>
  Promise.resolve({
    user,
    userProfileUrl: process.env.NEXT_PUBLIC_IGRP_PROFILE_URL || "",
    // ...
  }),
getSidebarData: () =>
  Promise.resolve({
    menuItems: menu,
    user,
    // ...
  }),
```

Also used object shorthand (`user` instead of `user: user`, `apps` instead of `apps: apps`).

### Migration Step SLMDF

1. In `src/igrp.template.config.ts`, inside `layoutMockData`:
   - Replace `async () => ({...})` with `() => Promise.resolve({...})`
   - Use object shorthand where applicable

## Extract Login Path Helper

### Overview ELPH

Login path construction in `(igrp)/layout.tsx` was verbose. A reusable helper was added to `sanitize.ts`.

### Changes Made ELPH

#### 1. New Helper (`src/lib/sanitize.ts`)

**Added:**

```typescript
export function getLoginPath(baseUrl: string, path = "/login"): string {
  try {
    return new URL(path, baseUrl).pathname;
  } catch {
    return "/login";
  }
}
```

#### 2. Layout Update (`src/app/(igrp)/layout.tsx`)

**Before:**

```typescript
const resolvedBaseUrl = baseUrl || "http://localhost:3000";
let loginPath: string;
try {
  loginPath = new URL(urlLogin || "/", resolvedBaseUrl).pathname;
} catch (error) {
  logger.error("[Layout] Invalid URL construction", error, { ... });
  loginPath = "/login";
}
```

**After:**

```typescript
const baseUrl = process.env.NEXTAUTH_URL_INTERNAL || process.env.NEXTAUTH_URL || "http://localhost:3000";
const urlLogin = "/login";
const loginPath = getLoginPath(baseUrl, urlLogin);
```

### Migration Steps ELPH

1. In `src/lib/sanitize.ts`, add the `getLoginPath` function (with JSDoc) before `sanitizeRedirectUrl`.

2. In `src/app/(igrp)/layout.tsx`:
   - Add `import { getLoginPath } from "@/lib/sanitize";`
   - Remove `import { logger } from "@/lib/errors";` if no longer used
   - Replace the try/catch block with `getLoginPath(baseUrl, urlLogin)`

## Web Interface Guidelines – Accessibility & UX

### Overview WIG

Changes to align with Web Interface Guidelines: accessibility, animations, typography, and anti-patterns.

---

### 1. Config (`src/config/login.ts`)

**Change:** `loginButton` set to `"Autenticar"` to match the form and language.

---

### 2. Logout Page (`src/app/(auth)/logout/page.tsx`)

**Before:** `<div>Logout in progress</div>`

**After:**

- Wrapped in `<main id="main-content" aria-live="polite" aria-busy="true">`
- Screen-reader-only `<h1>Logout</h1>`
- Loading text: `"A terminar sessão…"` (proper ellipsis `…`)

---

### 3. Login Page (`src/app/(auth)/login/page.tsx`)

**Changes:**

- `aria-label="Página de início de sessão"` on the section
- Skip link "Saltar para o conteúdo" with `sr-only` and `focus-visible` styles (visible on focus)
- `id="main-content"` and `scroll-mt-4` on the main content target
- Carousel loading skeleton: `motion-reduce:animate-none motion-safe:animate-pulse` and `aria-hidden="true"`

---

### 4. Home Page (`src/app/(igrp)/page.tsx`)

**Before:** `<div className="text-3xl font-bold">IGRP NEXT.js Template</div>`

**After:**

- Wrapped in `<main id="main-content">`
- Content inside `<h1>`

---

### 5. Globals CSS (`src/styles/globals.css`)

**Added in `@layer base`:**

```css
@media (prefers-reduced-motion: reduce) {
  *,
  *::before,
  *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**Note:** `@media` cannot be used inside `@theme` (Tailwind v4). It must be placed in `@layer base` or a regular block.

---

### 6. Auth Form (`packages/framework/next-ui/src/components/auths/form.tsx`)

**Changes:**

- Loading text: `"A autenticar..."` → `{texts.loginButton}…` (fix typo, use proper ellipsis)
- Use `texts.loginButton` for both states (configurable)
- Error alert: `role="alert"` and `aria-live="polite"`
- Icons (AlertCircle, Loader, ShieldCheck): `aria-hidden`
- Loader: `motion-reduce:animate-none`
- Button: `aria-busy={isLoading}` instead of `aria-live="polite"`

---

### 7. Auth Carousel (`packages/framework/next-ui/src/components/auths/carousel.tsx`)

**Changes:**

- **Anti-pattern fix:** `transition-all` → `transition-[width,background-color] duration-350 delay-350`
- Auto-advance respects `prefers-reduced-motion` and skips when `carouselItems.length <= 1`:

  ```typescript
  const prefersReducedMotion = typeof window !== 'undefined' &&
    window.matchMedia('(prefers-reduced-motion: reduce)').matches;
  if (prefersReducedMotion || carouselItems.length <= 1) return;
  ```

- `intervalTime` made configurable and added to `useEffect` deps

## Notes

- `getConfig()` uses React `cache()` so it is computed once per request when used by both layouts.
- Dynamic server usage errors must be rethrown; they indicate routes that cannot be statically generated.
- Skip links require `focus-visible` overrides to override `sr-only` and make the link visible when focused.
- `prefers-reduced-motion` in `@layer base` affects all animations and transitions; avoid using it inside `@theme`.
